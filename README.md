## ✨ 核心特性：AI 智能运维引擎

- 🤖 **零玩家感知自动升级**：内置原生纯异步 RCON 引擎。检测到 Mod 更新时自动窥探服务器人数，**0 玩家在线**则触发“静默保存关机”，配合 Docker 自动重启完成无痛升级；**有玩家在线**则推迟动作并下发中文广播，绝不背刺玩家！
- 📡 **游戏内 RCON 级联动**：彻底打通 QQ 群与游戏内屏幕！支持跨次元下发全服中文广播，在执行强制重启前给予玩家安全下线倒计时。
- 🏥 **军工级服务器健康探针 (TCP Ping)**：每次服务器重启（手动或自动）后，进入双重健康等待校验期。若探测到游戏端口（如 27015）恢复在线，全自动清零红灯并发送早报；若探测失败启动延时重试，死机则疯狂报警群主。
- 📊 **防刷屏级展示面板**：使用 QQ 原生“合并转发”卡片展示数百个模组的精确到分钟的时间戳与健康状态，告别霸屏烦恼。
- ⏰ **宿主机时钟同步**：完美解耦！支持设定每日定时静默安检（配合宿主机底层 Cron 定时重启使用），每日清晨自动抹平旧账簿。
- 🌐 **代理穿透与防阻断**：原生接管 Docker 环境变量代理（`trust_env=True`），自带重试机制，无缝绕过国内直连 Steam API 的网络墙。

## 📦 安装与初始化

1. **通过 AstrBot 后台安装**：在 AstrBot 的 Web 管理面板的“插件市场 / 自定义安装”中，输入本 Github 仓库地址进行安装。
2. **底层环境准备 (核心)**：
   - 确保你的游戏服务端开启了 RCON 并**设置了密码**。
   - 确保你的服务端容器（Docker）开启了 `always` 或 `unless-stopped` 的自动重启策略。
   - 确保游戏主机的 RCON 端口（默认 TCP 27015）对 AstrBot 所在网络互通。

## 💻 交互指令字典

支持全流程的群内交互，让服主在手机 QQ 上即可掌控全局。使用前缀 `/steammod`：

| 指令 | 权限 | 动作反馈 |
| :--- | :--- | :--- |
| `/steammod` | 所有人 | 📊 **查表**：调出合并转发卡片，查阅全服模组最新状态与时间戳。 |
| `/steammod ping` | 所有人 | 📡 **探针**：立即对游戏服务器发起 TCP 端口嗅探，判断服务器真实存活状态。 |
| `/steammod restart` | 管理员 | ⚡ **处决**：无视人数，强制下发【60秒】倒计时广播 ➡️ 触发 `save` ➡️ 触发 `quit` ➡️ 等待重启后进行健康校验。 |
| `/steammod reset` | 管理员 | 🔄 **擦除**：强制人工清空当前记录的所有红灯报警。 |
| `/steammod on/off`| 管理员 | ▶️ **启停**：手动启动/挂起后台的静默轮询更新任务。 |

## ⚙️ 可视化配置全景指南

在 AstrBot Web 管理后台中，本插件提供了极度自由的配置项（改动即时生效）：

### 基础与网络设置
| 配置项 | 描述 |
| :--- | :--- |
| `push_group_id` | **接收预警的 QQ 群号**。填写后插件即在后台工作。 |
| `mod_ids` | **模组 ID 列表**。支持从服务器 `.ini` 文件直接复制，自动切分过滤。 |
| `poll_interval_minutes`| **轮询间隔（分钟）**。静默检查更新的频率，建议 `30`。 |

### RCON 与探针配置 (高级联动核心)
| 配置项 | 描述 |
| :--- | :--- |
| `server_ip` | **服务器 IP/域名**。用于探针和 RCON 连接。 |
| `server_port` | **RCON 端口**。必须是 TCP 端口（PZ 默认 `27015`）。 |
| `server_rcon_password` | **RCON 密码**。解锁“在线人数检测”和“执行安全关机”的钥匙。 |
| `server_restart_wait_minutes`| **探针延迟时间(分钟)**。发送重启指令后，等待多久再进行首次 Ping 校验（建议 `6` 分钟）。 |

### 极客文案与广播设定
| 配置项 | 描述 |
| :--- | :--- |
| `server_rcon_broadcast` | **温柔提醒广播**：检测到更新且有玩家时的游戏内弹字（如：“模组有更新，不影响本次游玩，稍后将自动升级...”）。 |
| `server_rcon_countdown` | **零玩家秒退等待**：确认 0 玩家后，缓冲多少秒执行关机（建议 `5`）。 |
| `server_rcon_manual_broadcast`| **人工处决广播**：使用 `/steammod restart` 时发出的最后通牒。 |
| `server_rcon_manual_countdown`| **人工处决倒计时**：发出警告后留给玩家逃生下线的时间（建议 `60`）。 |

### 定时保底机制 (每日维护)
| 配置项 | 描述 |
| :--- | :--- |
| `auto_reset_enable` | 是否开启每日清晨的健康自检。 |
| `auto_reset_time` | 建议设定在你的宿主机底层定时重启（如绿联 cron）之后的 15 分钟（如 `04:20`）。 |

## 🧠 AI 智能工作流解析 (它是如何思考的？)

**【场景 A：深夜无玩家在线时发现更新】**
1. 监控线程捕获版本号变化。
2. 调用 RCON 探测在线人数 `Players = 0`。
3. 拦截 QQ 群刷屏，发送静默升级预警。
4. 执行极速 Save & Quit。
5. 挂起等待 6 分钟后，探针长驱直入。发现端口存活，清空红灯，QQ 群报喜：“✅ 重启并拉取模组成功！”

**【场景 B：晚高峰玩家激战时发现更新】**
1. 监控线程捕获版本号变化。
2. 调用 RCON 探测在线人数 `Players = 4`。
3. 挂起重启流程。跨次元向游戏内发送设定好的中文广播，安抚玩家。
4. 向 QQ 群发送报告：“发现更新，当前有 4 人在线，已推迟升级。”
5. 在下一次轮询（30分钟后），若发现玩家归零，瞬间切入【场景 A】完成收尾！

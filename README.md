
# 🎮 AstrBot Plugin: Steam Mod Monitor (Steam 创意工坊管家与游戏服中控)

**当前版本**: `v5.4.2`
**适用框架**: [AstrBot](https://www.google.com/search?q=https://github.com/Soulter/AstrBot)
**核心受众**: 《僵尸毁灭工程》(Project Zomboid) 及各类支持 Source RCON 协议的游戏服主。

本插件是一个极其硬核且高度自动化的服务器中控系统。它不仅能 24 小时替你盯着 Steam 创意工坊的 Mod 更新，还能通过 RCON 和 SSH 双通道，实时掌控游戏服务器的物理底层和游戏内状态，实现**“有人预警，无人静默重启更新”**的完美运维闭环！

---

## ✨ 核心特性 (v5.4.2)

* **🕵️‍♂️ 智能 Mod 嗅探与更新**：定时调用 Steam Web API 检查订阅的 Mod 列表。发现更新时，如果服务器空闲，自动断电/调用后台脚本拉取更新；如果有人在线，则向游戏内发送全局广播预警。
* **📊 终极全景体检报告 (`/steammod check`)**：一键生成精美排版的中控仪表盘。
* **SSH 物理探测**：无视语言环境，精准抓取宿主机的 CPU负载、内存占用率、硬盘可用空间（兼容 LVM 逻辑卷）、系统运行时间。
* **核心备份监控**：自动识别 `/home/steam/` 下的最新 `.tar.gz` 备份档案，校验生成时间与文件大小。
* **RCON 游戏态势**：获取实时在线人数，并提取当前在线玩家的完整 ID 名单。


* **🔌 双模重启引擎 (Dual-Engine Restart)**：
* `rcon` 模式：通过 RCON 发送保存和关机指令，极其轻量。
* `linuxgsm` 模式：底层直连，通过 SSH 在独立终端内执行 `./pzserver restart`，完美兼容 LinuxGSM 框架，拒绝内存配置覆盖死锁。


* **🛡️ 每日防爆自检**：自定义每日静默安检时间（如 04:20），主动探测 TCP 端口存活状态，防止僵尸进程。

## 🛠️ 安装说明

1. 将本插件文件夹放入 AstrBot 的 `data/plugins/` 目录下。
2. **环境依赖**：如果你打算使用 `linuxgsm` 模式（SSH 底层控制）和 `/steammod check` 的底层物理探测功能，请必须在 AstrBot 运行环境中安装 `asyncssh` 库：
```bash
pip install asyncssh

```


3. 重启 AstrBot 框架。

## ⚙️ 核心配置说明 (通过 AstrBot 管理面板填写)

在 AstrBot 的 Web 管理面板中，找到本插件并填写配置：

| 配置项 | 说明 | 示例 / 建议 |
| --- | --- | --- |
| **推送群号** | 接收更新通知和体检报告的 QQ 群号。 | `123456789` |
| **模组 ID 列表** | 纯数字 ID，用逗号或分号隔开。 | `3077900375;2896041179` |
| **API 接口地址** | 默认官方 API，国内被墙可换反代。 | `https://api.steampowered.com` |
| **服务器 IP & 端口** | 游戏服的公网 IP 和 **RCON 端口**。 | `1.2.3.4`, `27015` (注意不是16261) |
| **RCON 密码** | 必须填写，用于发广播、踢人和取名单。 | `YourStrongPassword` |
| **自动更新倒计时** | 无人在线时，执行保存并关机前的缓冲秒数。 | `5` |
| **人工强制重启倒计时** | 执行强制重启指令后，留给玩家的逃生时间。 | `60` |
| **重启/更新执行方式** | 选 `rcon` 或 `linuxgsm`。 | 推荐使用 `linuxgsm` |
| **SSH 登录信息** | 仅 `linuxgsm` 模式及 `check` 指令需要。 | 填入虚拟机的 IP、端口(22)、账密 |

## 💻 交互指令指南

在配置好的 QQ 群内，直接发送以下指令即可操控服务器：

* **`/steammod check`** 🌟 (核心功能)
生成【服务器全景运行体检报告】。包含 SSH 物理状态（CPU/内存/硬盘/备份）、RCON 游戏状态（进程存活/在线人数/玩家名单）及管家健康度。
* **`/steammod`**
手动查验当前配置的所有 Mod 最新状态（是否与内存记录一致）。
* **`/steammod restart`**
强制重启指令！向游戏内发送红色广播预警，倒计时结束后强制保存并重启服务端。
* **`/steammod ping`**
轻量级网络探针，仅测试游戏服 TCP 端口是否通讯正常。
* **`/steammod on`** / **`/steammod off`**
热挂起或恢复 Mod 自动轮询监控功能。
* **`/steammod reset`**
清理内存中的“待更新黑名单”，强制将所有指示灯重置为绿灯（适用于人工解决冲突后）。

## ⚠️ 常见问题与排雷指南

 **执行 `/steammod check` 时物理状态全是“获取失败”？**
* 请检查是否安装了 `asyncssh` 库。
* 请检查 AstrBot 管理面板中的 `SSH 主机 IP`、`用户名`、`密码` 是否填写正确。
* 请检查 Ubuntu 虚拟机的系统时区是否为 `Asia/Shanghai` (`CST`)，否则 Cron 定时任务会按照 UTC 时间错误执行。

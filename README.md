
# 🎮 AstrBot Plugin: Steam Mod Monitor (Steam 创意工坊管家)

**当前版本**: `v5.4.2`
**适用框架**: [AstrBot]

本插件是一个功能高度弹性的 Steam 创意工坊监控工具。无论你是单纯想追踪心仪 Mod 更新动态的**普通玩家**，还是需要 24 小时无人值守、全自动更新游戏服务器的**硬核服主**，本插件都能为你提供极其完美的解决方案。

---

## 🌟 维度一：日常订阅玩家 (轻量化 Mod 监控)

对于普通游戏玩家或小群友来说，你可能并没有开服务器，只是想在某个长期关注的 Mod（比如某个大型汉化包、某个长期鸽置的坑）更新时，第一时间收到 QQ 群通知。

### ✨ 核心功能

* **🕵️‍♂️ 无感静默嗅探**：定时调用 Steam Web API，精准比对你订阅的 Mod 列表最后更新时间。
* **📢 QQ群实时推送**：一旦发现作者更新，立刻向指定的 QQ 群发送精美的更新播报日志。
* **🔍 手动查验**：群友随时可通过指令核对所有被监控 Mod 的当前健康状态。

### 🛠️ 极简配置 (仅需 3 项)

在 AstrBot 控制面板中，你只需要填写以下几项即可完美运行基础功能：

1. **推送群号**：填入接收更新通知的 QQ 群号。
2. **模组 ID 列表**：填入你想监控的 Steam 创意工坊纯数字 ID，用逗号或分号隔开（例如：`3077900375;2896041179`）。
3. *(可选)* **API Key**：填入 Steam Web API Key 以防查询过快被 Steam 限制频率。

## 👑 维度二：硬核游戏服主 (全自动化运维中控)

对于《僵尸毁灭工程 (Project Zomboid)》及各类支持 Source RCON 协议的游戏服主来说，本插件是一个极其硬核的**“服主级中控仪表盘”**。它打通了 RCON 与 SSH 双通道，实现了**“有人预警，无人静默重启更新”**的完美闭环。

### ✨ 独家硬核特性 (v5.4.2 最新增强)

* **🤖 智能防踢重启引擎**：
* **无人在线**：检测到 Mod 更新且服务器无活人时，自动发送保存指令并静默重启更新。
* **有人在线**：绝不强踢！改为向游戏内发送全局广播预警（如：*“有Mod更新，请在安全区等待重启”*），等待玩家安全下线后再执行。


* **📊 终极全景体检报告 (`/steammod check`)**：
* **SSH 底层物理探测**：无视 Linux 中英文语言环境差异，精准抓取宿主机的 CPU 负载、内存占用率、硬盘可用空间（完美兼容 LVM 逻辑卷）。
* **核心备份监控**：自动识别 `/home/steam/` 下的最新的 `.tar.gz` 备份档案，校验其是否为今日生成及文件大小。
* **RCON 游戏态势**：瞬间抓取实时在线人数，并提取当前在线玩家的完整 ID 名单。


* **🔌 双模安全重启 (Dual-Engine)**：
* `rcon` 模式：通过 RCON 发送游戏内保存和关机指令。
* `linuxgsm` 模式：底层直连，通过 SSH 在独立终端内执行 `./pzserver restart`，完美兼容 LinuxGSM 框架，**拒绝服务端“关机内存覆写”导致的沙盒设置失效死锁**。



### 🛠️ 进阶配置依赖

除了基础的群号和 Mod ID 外，服主需要配置：

1. **RCON 设定**：填入服务器 IP、RCON 端口（默认 27015）及 RCON 密码。
2. **SSH 设定**：填入虚拟机的 IP、端口(22) 及系统账密（仅 `linuxgsm` 模式和 `check` 指令需要）。
3. **环境依赖**：必须在 AstrBot 运行的 Python 环境中安装异步 SSH 库：
```bash
pip install asyncssh

```

## 💻 交互指令指南 (QQ群内使用)

| 指令 | 适用维度 | 功能说明 |
| --- | --- | --- |
| **`/steammod`** | 全员可用 | 核对并返回当前所有被监控 Mod 的最后更新时间及状态。 |
| **`/steammod on/off`** | 管理员 | 热挂起或恢复 Mod 的后台自动轮询监控功能。 |
| **`/steammod check`** | 👑 服主专属 | 🌟生成**【服务器全景运行体检报告】**（物理状态+游戏状态+管家状态+备份监控）。 |
| **`/steammod restart`** | 👑 服主专属 | 手动发送强制重启指令！向游戏内下发红色倒计时广播，随后强制保存关机并拉取更新。 |
| **`/steammod ping`** | 👑 服主专属 | 轻量级网络探针，仅测试游戏服 TCP 端口是否通讯正常。 |
| **`/steammod reset`** | 👑 服主专属 | 清理内存中的“待更新黑名单”，强制将所有报警红灯重置为绿灯（适用于人工解决冲突后）。 |

## ⚠️ 常见问题与排雷指南

**Q: 执行 `/steammod check` 时物理状态全是“获取失败”？**

> **A:** 请检查 Python 环境是否安装了 `asyncssh` 库；并在 AstrBot 管理面板中核对 `SSH 主机 IP`、`用户名`、`密码` 是否填写正确。


**Q: 为什么生成的备份时间和实际时间差了 8 个小时？**

> **A:** 典型的 Linux 时区陷阱。Ubuntu 默认使用 UTC（零时区），请通过 `sudo timedatectl set-timezone Asia/Shanghai` 将系统修改为北京时间，并使用 `sudo systemctl restart cron` 重启定时服务。

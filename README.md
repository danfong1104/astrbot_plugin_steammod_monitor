
# 🎮 AstrBot Steam 创意工坊模组监控插件 (Steam Mod Monitor)

> 专为硬核游戏服主（僵尸毁灭工程、方舟、Rust等）打造的“赛博网监”级 AstrBot 插件。
> 彻底告别“玩家进不去服务器才发现模组更新”的窘境，实现全自动更新预警与优雅的群组推送！

## ✨ 核心特性

- 🚦 **防刷屏级展示面板**：使用 QQ 原生“合并转发（聊天记录）”卡片展示数百个模组的健康状态，告别霸屏烦恼。
- 🕒 **硬核时间戳查证**：每一条模组状态都带有 Steam 官方记录的精确更新时间，绝不谎报军情。
- 📋 **极度舒适的配置体验**：完美兼容服务端 `.ini` 文件！模组 ID 支持使用逗号 `,` 或分号 `;` 分隔，直接一键全选复制粘贴即可。
- 🛡️ **抗网络波动机制**：自带可配置的最大重试次数（默认 5 次），无惧 Steam 间歇性抽风。
- 🌐 **代理穿透支持**：原生接管 Docker 环境变量代理（`trust_env=True`），无缝绕过国内直连 Steam API 的网络墙。
- 🧹 **热重载无残留**：彻底解决 AstrBot 框架重启插件时产生的“幽灵僵尸任务”导致的重复推送问题。

## 📦 安装与初始化

1. **通过 AstrBot 后台安装**：
   在 AstrBot 的 Web 管理面板的“插件市场 / 自定义安装”中，输入本 Github 仓库地址进行安装。
2. **依赖检查**：
   本插件依赖高性能异步网络库 `aiohttp`，安装插件时 AstrBot 会自动通过 `requirements.txt` 安装。
3. **初始配置**：
   安装完成后，请在 Web 面板的【插件配置】中填写**推送群号**和**模组 ID 列表**。

## ⚙️ 配置说明

在 AstrBot Web 管理后台中，你可以自定义以下选项：

| 配置项 | 描述 | 默认值 / 建议 |
| :--- | :--- | :--- |
| `push_group_id` | **接收预警的 QQ 群号**。填写后插件启动即自动在后台开始轮询。 | 必填项 |
| `mod_ids` | **模组 ID 列表**。支持逗号 `,` 或分号 `;` 隔开。 | 可直接从服务端配置文件粘贴 |
| `poll_interval_minutes` | **轮询间隔（分钟）**。向 Steam 查询的频率。 | 建议 `30` |
| `max_retries` | **请求失败重试次数**。网络波动时的挣扎次数。 | 默认 `5` 次 |
| `steam_api_base` | **API 接口地址**。如果国内服务器无法直连，可修改为反代地址。 | `https://api.steampowered.com` |
| `steam_api_key` | **API Key (可选)**。填入可大幅降低被限制请求频率的概率。 | 建议前往 Steam 官网申请 |
| `push_template` | **群内推送预警模板**。支持占位符 `{mod_name}` 和 `{mod_id}`。 | 自定义文本，支持艾特全体 CQ 码 |

## 💻 交互指令

所有的群内交互都高度收拢在 `/steammod` 指令下，保持群聊环境整洁：

| 指令 | 权限 | 功能描述 |
| :--- | :--- | :--- |
| `/steammod` | 所有人 | 📊 **查表**：召唤“合并转发卡片”，列出所有模组的红绿灯状态与最后更新时间戳。 |
| `/steammod reset` | 所有人 | 🔄 **重置**：服主重启游戏服务器（拉取完最新模组）后使用，将所有 🔴红灯 消除变回 🟢绿灯。 |
| `/steammod on` | 管理员 | ▶️ **开启**：手动启动后台静默轮询预警任务。 |
| `/steammod off` | 管理员 | ⏸️ **暂停**：手动关闭后台静默轮询。 |

*(注：如果配置了正确的群号和模组ID，插件在重启 AstrBot 时会自动执行 `on` 逻辑，无需每次手动开启)*

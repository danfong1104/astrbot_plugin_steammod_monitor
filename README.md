## ✨ 核心特性：双模式自由切换

本插件采用“渐进式增强”设计，按需配置，丰俭由人。

### 🛠️ 基础模式（面向所有玩家 & 游戏群）
无需任何服务器权限，开箱即用的 Steam 创意工坊追踪器。
- 🔍 **全自动静默轮询**：后台自动比对数十/上百个 Steam 创意工坊 Mod 的最新时间戳。
- 📢 **精准更新推送**：发现更新后，自动向指定 QQ 群推送通知（“检测到steam订阅模组更新”）。
- 📊 **防刷屏状态面板**：使用原生合并转发卡片，群友随时发送指令即可查阅所有 Mod 的红绿灯健康度与最后更新时间。
- 🌐 **自带网络穿透**：底层自带异步重试与环境变量代理支持，无缝绕过 Steam API 直连的网络波动。

### 👑 进阶服主模式（面向游戏服主，如僵尸毁灭工程、方舟等）
填入服务器 IP 与 RCON 密码，立刻解锁“赛博级服主 AI 管家”。
- 🤖 **零玩家感知无痛升级**：发现 Mod 更新时，自动通过 RCON 嗅探在线人数。**无人在线**则静默触发安全保存与关机（配合 Docker 自动重启完成无痛更新）；**有玩家在线**则推迟动作并下发游戏内中文广播，绝不背刺玩家！
- 🏥 **军工级健康探针 (TCP Ping)**：每次服务器重启（手动或自动）后，进入双重健康等待校验期。探测到端口存活则自动清零红灯并发送群报喜；死机则疯狂报警。
- 📡 **跨次元 RCON 联动**：支持跨次元下发全服中文广播，在执行强制重启前给予玩家安全下线倒计时。
- ⏰ **宿主机时钟同步清零**：支持设定每日定时静默安检（配合宿主机底层 Cron 定时重启使用），每日清晨自动抹平旧账簿。

## 💻 交互指令字典

支持全流程的群内交互，让服主和玩家在手机 QQ 上即可掌控全局。使用前缀 `/steammod`：

| 指令 | 权限 | 动作反馈 |
| :--- | :--- | :--- |
| `/steammod` | 所有人 | 📊 **查表**：调出合并转发卡片，查阅订阅列表内所有 Mod 的最新状态与时间戳。 |
| `/steammod ping` | 所有人 | 📡 **探针**：立即对游戏服务器发起 TCP 端口嗅探，判断服务器真实存活状态（需配置 IP）。 |
| `/steammod restart` | 管理员 | ⚡ **处决**：无视人数，强制下发【60秒】倒计时广播 ➡️ 触发存档 ➡️ 安全关机重启。 |
| `/steammod reset` | 管理员 | 🔄 **擦除**：强制人工清空当前记录的所有红灯报警。 |
| `/steammod on/off`| 管理员 | ▶️ **启停**：手动启动/挂起后台的静默轮询更新任务。 |

## ⚙️ 极简配置指南

在 AstrBot Web 管理后台中安装本插件后，即可进行可视化配置：

### 🟢 基础版必填（仅用作更新追踪）
- `push_group_id`: **接收预警的 QQ 群号**。填写后插件即在后台工作。
- `mod_ids`: **模组 ID 列表**。纯数字，用逗号或分号隔开。支持直接从服务端配置文件中复制粘贴。
- `poll_interval_minutes`: **轮询间隔（分钟）**。建议设为 `30`，避免被 Steam 限制请求频率。
- `steam_api_key`: (可选) 填入你的 Steam Web API Key 可大幅提升请求稳定性。

### 🔴 进阶服主选填（解锁自动重启与探针）
如果你拥有自己的游戏服务器，并希望插件帮你全自动更新 Mod，请继续填写以下内容：
- `server_ip` & `server_port`: 你的服务器 IP 与 TCP 端口（建议填 RCON 端口，如 PZ 的 27015）。
- `server_rcon_password`: 服务器的 RCON 管理员密码（解锁“人数检测”和“安全关机”的钥匙）。
- `server_restart_wait_minutes`: **探针延迟时间(分钟)**。执行重启后，等待多久再进行首次 Ping 校验（建议根据服务器开服加载速度设置，如 `6` 分钟）。
- `auto_reset_enable` & `auto_reset_time`: 开启每日清晨的健康自检。建议设定在你的宿主机底层定时重启之后的 15 分钟（如 `04:20`）。

> **💡 提示**：你可以自由定制游戏内发送的 RCON 中文广播预警文本，详情请在后台查看“广播设定”相关配置项。

## 🧠 AI 智能工作流解析 (它是如何思考的？)

**【场景 A：深夜无玩家在线时发现更新】**
1. 监控线程捕获 Steam 创意工坊版本号变化。
2. 调用 RCON 探测在线人数 `Players = 0`。
3. 拦截 QQ 群刷屏，发送静默升级预警。
4. 执行极速 Save & Quit。
5. 挂起等待设定好的开服时间（如 6 分钟）后，探针长驱直入。发现端口存活，清空红灯，QQ 群报喜：“✅ 服务器连通性检测正常！最新模组已加载完毕...”

**【场景 B：晚高峰玩家激战时发现更新】**
1. 监控线程捕获版本号变化。
2. 调用 RCON 探测在线人数 `Players = 4`。
3. 挂起自动重启流程。跨次元向游戏内发送设定好的中文广播，温柔提醒玩家。
4. 向 QQ 群发送报告：“发现更新，当前有 4 人在线，已推迟升级。”
5. 在下一次轮询（30分钟后），若发现玩家归零，瞬间切入【场景 A】完成收尾！
